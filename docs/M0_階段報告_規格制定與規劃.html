1執行摘要

目標:建立一套 DCC(離焦轉換係數)校正工具,對單顆 PDAF 相機模組,由 10 點鏡頭掃描之逐區相位視差回歸出 8×6 DCC map,經 focus 交叉驗證後打包為可燒錄之校正 block。

本階段最重要的架構決議(2026-07-11):前期為離線工具。 實機擷取、PD 像素處理與 SAD disparity 計算、EEPROM 實體燒錄,全部由外部模組承擔; 本工具消費外部產出的「逐區 disparity/focus 序列」,專注於回歸、驗證、打包、報告, 所有硬體與演算法介面保留,待 M2 對接。

跨團隊認知對齊 — 一句話版本 外部團隊交付 disp_seq.json(格式見 §5,精度要求見 §8),本工具產出 report.json / report.md / block.bin;兩者之間的一切數值以 raw_pixel 為 disparity 單位、 DAC/raw_pixel 為 DCC 單位,任何偏離都會造成 16 倍等級的對焦錯誤。

3校正原理與數學模型

3.1 物理模型

PDAF 的左/右(L/R)相位像素觀察同一景物,離焦時產生水平視差(disparity)。 在合焦點附近,視差與鏡頭位置(VCM DAC 碼)呈線性關係。DCC 即此線性關係的斜率:

(3-1)DAC = k · disp + b, DCC ≡ k = ΔDAC / Δdisparity [DAC / raw_pixel],恆正

其中截距 b 有明確物理意義:disparity = 0 時的鏡頭位置,即合焦點。 因此可用另一條獨立途徑——focus 曲線(對比度/梯度能量)的峰值位置——來交叉驗證回歸結果:

(3-2)err = | b − focus_peak | / (NEAR − FAR), err < tolerance(metal-shield 預設 0.20)→ PASS
認知對齊 ②:三個不變的方向約定
  • 掃描方向:FAR→NEAR 單向,禁止反轉(VCM 磁滯對策)。序列檔的 dacs 必須嚴格遞增。
  • disparity 正負:R 影像相對 L 影像右移為正。
  • DCC 恆正:回歸斜率 k ≤ 0 即報 E-E01(最常見根因:L/R 定義顛倒)。

5介面契約 ①:disparity/focus 序列(disp_seq.json)

這是外部 SAD 模組 → 本工具的唯一量測資料介面,也是目前唯一待跨團隊確認的依賴。 每模組一份 data/disparity/<module_id>/disp_seq.json(UTF-8):

欄位型別必填約束
module_idstr✔與資料夾名一致
unit"raw_pixel" | "pd_image_grid"✔須等於 config input_disparity_unit
pitch_xint✔須等於 config 推得之 pitch_x(範例 16;單位驗算用)
dacs[int × N]✔N == num_positions(v1: 10);嚴格遞增(FAR→NEAR);與 C-1 規劃逐點一致 ±1 DAC
grid_w / grid_hint✔外部 SAD 之輸出粒度,可 ≠ 8×6(如 pattern block 級 144×108);不一致時本工具依 §7 聚合
data[N][grid_h][grid_w] float | null✔disparity;無效樣本以 null(品質門檻判定在外部完成)
focus[N][grid_h][grid_w] float✔逐 cell focus value(梯度能量等)。Phase F 只用「峰值位置」,量綱不拘但同序列內須一致
quality同 data 形狀選填品質指標;weighted_mean 聚合時作權重(缺則等權)
metadict選填曝光、溫度、治具 ID 等追溯資訊
認知對齊 ③:focus value 必填的原因 交叉驗證(Phase F)需要每張、每區的 focus value 來擬合峰值。像素處理已外部化, 本工具無法自行計算——外部模組必須隨 disparity 一併輸出 focus 序列,否則整個 PASS/FAIL 判定無法執行。
給外部 SAD 模組團隊的三件事
  1. 格式:照上表輸出 disp_seq.json(自述式,含單位與粒度欄位)。
  2. 精度:單點 disparity |系統偏差| ≤ 0.3 raw px、隨機 σ ≤ 0.5 raw px(推導見 §8;此為介面驗收條件)。
  3. 品質責任切分:SAD 品質門檻(曲線深度/對比等)由外部判定,不合格 cell 直接填 null;本工具只統計有效樣本數並聚合。

6介面契約 ②:單位契約與 16 倍陷阱

本專案風險最高的認知差異點。PD 影像(288×432)的座標系中「1 格」= 原始 sensor 的 pitch_x = 16 個 raw pixel。同一個物理視差,兩種座標系的數值差 16 倍,DCC 隨之差 16 倍—— 單位搞錯,runtime 一步跳焦會過衝或幾乎不動 16 倍。

量raw_pixel 座標系pd_image_grid 座標系關係
同一物理視差16.0 px1.0 格×16
DCC(中央)12.46 DAC/px199.36 DAC/格×16
Q6 編碼797 = 0x031D12759 = 0x31D7皆在 uint16 範圍內
認知對齊 ④:runtime PDLIB 的單位仍待確認(開放問題 #2,延期至 M2 燒錄前) 設計已對沖:core 永遠 raw_pixel、打包端兩條路徑都實作並測試(UT-09)、report 同時記錄兩種單位的 DCC。 PDLIB 單位確認後只翻一個 config 值(output_disparity_unit),不改程式。

7介面契約 ③:粒度聚合規則(Phase D-5)

外部 SAD 模組的輸出粒度不必等於 DCC 的 8×6 分區(例如以 32×32 pattern block 為單位輸出 144×108 個 cell)。 本工具在 Phase D-5 依 config 聚合:

規則定義
歸區以 cell 中心座標對 8×6 等分切割歸區;落在邊界者歸左/上區(確定性,無二義)
aggregation.methodmedian(預設,抗離群)或 weighted_mean(以 quality 為權重,缺 quality 則等權);NaN cell 不計
aggregation.min_valid_ratio預設 0.5;區內有效 cell 比例低於此 → 該區該幀無效(NaN),進入 D-6 有效樣本統計
一致性驗收UT-10:無雜訊細粒度序列經兩種方法聚合,皆須與直接 8×6 序列一致(<1e-9);0.49/0.50 邊界判定正確

8誤差預算之數學推導

目的:把系統級精度需求(重複性、DCC 準確度)反推為外部 SAD 模組單點 disparity 的精度上限, 作為可驗收的介面條件。推導全程以 config 參數符號表述,數字為預設 config 之推導示例。

8.1 已知條件與符號

符號意義預設 config 示例值
n掃描點數(num_positions)10
S = NEAR − FARsweep 全幅;S = span_af × (1 + far_margin + near_margin)400 × 1.2 = 480 DAC
h = S/(n−1)掃描步距480/9 ≈ 53.33 DAC
k真值 DCC(中央區)12.46 DAC/raw_px
σd單點 disparity 隨機誤差(iid,raw_pixel)——待求上限—

8.2 推導

步驟 1 — DAC 離散度。n 個等距點對其平均之平方和(等差級數之標準結果):

(8-1)Sxx = Σ(daci − d̄ac)² = h²·n(n²−1)/12 = 53.33² × 10×99/12 ≈ 2.35×10⁵ ⇒ √Sxx ≈ 484.4

步驟 2 — disparity 離散度。真值關係 disp = (dac − b)/k 為線性映射,故:

(8-2)√Sdd = √Sxx / k ≈ 484.4 / 12.46 ≈ 38.9 raw px

同時,全 sweep 的 disparity 動態範圍僅 ±S/(2k) ≈ ±19.3 raw px ≈ ±1.2 個 PD 影像格—— 這說明外部 SAD 的次像素內插品質是全案精度的第一決定因素(整數格搜尋幾乎不供獻資訊)。

步驟 3 — 斜率誤差傳遞。以無噪之 DAC 為自變數方向表述(擬合 disp 對 DAC 再反解,一階等價), 最小平方斜率估計的標準差:

(8-3)σk/k ≈ σd / √Sdd = σd · k / √Sxx = σd / 38.9

步驟 4 — 由各系統準則反推 σd 上限:

系統準則反推式σd 上限(示例)
NFR-02:重複性 CV < 2%σ_d ≤ 0.02·√Sxx/k≤ 0.78 raw px ← 綁定條件(margin=0 時收緊為 0.65)
IT-01:r² > 0.99σ_d ≤ √(0.01·Sdd/(n−2))≤ 1.37 raw px(最鬆,非綁定)
IT-01:中央 DCC 誤差 < 3%(含系統項)系統偏差另列預算|bias| ≤ 0.3 raw px(占比 < 1%)

步驟 5 — 回歸方向(errors-in-variables)自檢。實作採 DAC = k·disp + b 之一般 OLS, 雜訊在自變數會造成斜率衰減偏差,量級為:

(8-4)bias ≈ − σd² / (Sdd/n) = − 0.78² / 151.2 ≈ −0.40%

在預算內可忽略(<0.5%),故 v1 採一般 OLS 有數學依據;fitter 介面設計為可抽換, 精度需求提高時再換 EIV/反向擬合(開發備忘 #2)。

8.3 結論:對外部 SAD 模組的驗收要求

介面驗收條件(固定值,對 margin ∈ [0, 0.2] 全域保守) 於全 sweep 動態範圍內,單點 disparity: |系統偏差| ≤ 0.3 raw px(≈0.019×pitch) 且 隨機 σ ≤ 0.5 raw px(≈0.031×pitch)。
取值邏輯:以 margin=0 之最嚴綁定上限(0.65)再留裕度;margin 加大時裕度更寬。 本工具端以 IT-07(蒙地卡羅:注入 σ=0.5 + bias=0.3,×10 次,驗證 CV<2% 且偏差<3%)閉環驗證此預算成立。
歷史勘誤(避免引用舊文件) 早期草稿之 UT-03/04 容差(0.35×/0.25×pitch = 5.6/4 raw px)過鬆約一個數量級,已作廢。 若外部團隊手上有舊版數字,請以本節為準。

  • DAC 序列(margin 0.1):端點 180/660 精確、嚴格遞增、間距 53/54 交錯 ✔
  • 單位比值:dcc_pd_grid/dcc_raw_px = 199.36/12.46 = 16 = pitch_x ✔

11.1 合成資料(Sim 序列生成器)

直接生成 disp_seq.json,不經像素域。真值注入:true_dcc(r,c)(中央 12.46 → 角落 14.5)、 合焦 420 DAC;disp_true = (dac−420)/true_dcc;focus 為以 420 為峰之單峰曲線。 可注入:高斯雜訊 σd、系統 bias、null、DAC 亂序、單位錯置、細/粗粒度——正負向測試皆備。 無雜訊時解析真值 k = true_dcc、b = 420、focus_peak = 420(UT-05/06 準則依此)。

12決議紀錄與開放問題處置

12.1 本階段決議(2026-07-11)

  1. 離線前期範圍:擷取/SAD/燒錄外部化,介面保留(HAL 前期僅 Sim*)。
  2. 序列介面制定:disp_seq.json v1(含 focus value 必填、自述式粒度與單位)。
  3. 單位契約強化:disparity_unit 拆分為 input/output 兩參數;轉換僅限讀取端/打包端兩處;ratio_check 公式勘誤(恆等 pitch_x)。
  4. 合成真值對齊:UT-05/06 期望值修正為 b = peak = 420(消除與生成器定義的矛盾)。
  5. 誤差預算重推:舊 UT-03/04 容差作廢;改為外部驗收條件 bias ≤ 0.3 / σ ≤ 0.5 raw px + IT-07 閉環。
  6. margin 預設 0.1:連動處全面參數化(公式 + 推導示例),禁止寫死端點/分母。
  7. 粒度聚合:SAD 輸出粒度可設定;median(預設)/ quality 加權平均;min_valid_ratio 0.5。
  8. RAW 唯讀:僅作互動式 UI 底圖(FR-18),不入計算路徑、不得寫入。
  9. 回歸方法:v1 一般 OLS(EIV 偏差 ≈ −0.4% 可忽略);fitter 可抽換。
  10. 錯誤碼滾動制:初期允許通用碼,M1 出口前收斂;鐵律 4 漸進嚴格化。
