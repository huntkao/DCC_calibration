#鐵律證據
1單位契約(16 倍陷阱)轉換僅 disparity_reader(入)與打包端(出)兩處;UT-09 ratio_check 釘死漏乘/重複乘
2分層core 無外部依賴;依賴方向 gui/cli → app → io/hal → core,建置系統強制
3掃描方向 FAR→NEARsweep::plan 無反轉參數(VCM 磁滯)
4失敗落盤 + 具名錯誤碼IT-03 驗證 abort_dump.json;錯誤碼收斂為 E-A01/C01/D01..03/E01/F01/G01(M1 出口收斂完成)

分層(鐵律 2)與掃描方向(鐵律 3)是設計面保證——不是靠測試抓,而是介面本身無法違反(sweep::plan 沒有方向參數、core 不 link 任何 I/O)。這是「違反 = 編譯不過」而非「違反 = 測試紅」。
