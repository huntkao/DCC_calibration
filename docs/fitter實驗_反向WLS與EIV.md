# fitter 實驗:反向回歸、WLS 與 EIV(Deming)

> 日期:2026-07-17 · 設計:docs/superpowers/specs/2026-07-17-fitter-wls-eiv-design.md
> 資料:`dcc_cal --dry-run --fitter-scan`(50 seeds,focus_linked 誠實 quality)
> 對應:SPEC-005 §8 #2(EIV 偏差備忘)、SPEC-004 §3a.1(q→σ 標定)

## 1. 問題

回歸 DAC = k·disp + b 中 DAC 精確、disp 含噪 → 前向 OLS 斜率衰減
(理論 att = Sx²/(Sx²+σ²);UT-F2 已驗證理論式吻合)。

## 2. 方法

四 fitter(前向/反向/反向WLS/Deming)× σ ∈ {0.2, 0.5, 1.0, 2.0} × 50 seeds
配對比較(同 seed 同資料);q→σ 殘差標定(qsigma)每 σ 回推 p̂/σ̂₀。

殘差標定含槓桿修正:UT-Q2 與 `--fitter-scan` 的殘差收集皆對 n=10 擬合殘差
做 `|e| / sqrt(1 − h_ff)` studentized 修正(未修正時 hat matrix 收縮導致
p̂ 低估至 ≈0.24)。修正後 p̂ ≈ 0.40(仍略低於合成真值 0.5,studentized 在
異方差下僅為一階近似)→ γ̂ = 2p̂ ≈ 0.8;而下表顯示 WLS 對 γ 誤標並不敏感
(γ=0.5/2 之 CV 差距遠小於「用不用 WLS」的差距),故此殘差偏差不影響
WLS 消變異的實務結論。

註:p̂/r²/bins 四列相同是數學必然——p 為 log-log 斜率、對噪聲尺度不變,
同一 seed 集下噪聲只是等比縮放,不影響斜率估計;唯 σ̂₀(截距對應之尺度)
隨真實 σ 等比例上升,是本表唯一應隨 σ 變動的欄位。這不是 bug。

### 3.3 判讀

- 前向 OLS 偏差隨 σ² 放大:σ=0.5 → -0.0167%、σ=2.0 → -2.2957%(理論吻合;
  σ=0.2/0.5 時真實偏差訊號小於 50-seed 蒙地卡羅雜訊底,符號甚至不穩定,
  σ=1.0→2.0 之偏差比約 5.6 倍,與變異比 4 倍量級一致,訊號在大 σ 時才明顯
  浮現)。
- 反向消偏:σ=2.0 時 bias 1.3724%(vs 前向 -2.2957%)。反向並非零偏——
  純代數反轉引入自身正向偏移,量級小於前向但非零,此為預期行為
  (反向回歸把噪聲角色對調,並非「完美消偏」而是「偏差方向與量級改變」)。
- WLS(γ=1)相對等權反向之 CV 改善:9.33%(σ=0.2) → 8.99%(σ=2.0),
  四個 σ 上一致落在 9.0–9.3% 區間。
- γ 錯標靈敏度:γ=0.5/2 之 CV 差距在 3.1–4.7%(σ=2.0 時最小、σ=0.2 時最大),
  遠小於「用不用 WLS」帶來的 ~9% 改善(結論:對 γ 誤差不敏感)。
- Deming(δ=0)與反向逐列一致(位元級,UT 釘死)——主表中 deming 與
  ols_inverse 兩列在全部 4 個 σ 下數值完全相同,驗證通過。
- 額外發現(RMSE 綜合排名):wls_inverse(γ=1)在全部 4 個 σ 下 RMSE 最低
  (0.8960 / 2.2449 / 4.5229 / 9.3134),其次為 ols_forward,ols_inverse/deming
  最高。純反向雖消偏最徹底,但變異增加抵銷了消偏帶來的 RMSE 效益;
  WLS 同時降低變異且保留大半消偏效果,故综合精度最佳。
- qsigma 回推 p̂ ≈ 0.40(真值 0.5),σ̂₀ 誤差約 6.8%(四個 σ 下一致,因同一
  seed 集僅做等比縮放)——殘差標定程序可用於真實資料,惟需知悉 studentized
  近似帶來的系統性低估(見 §2)。

## 4. 結論與建議

**典型產線噪聲(§3a 預算 σ≤0.5)下,前向偏差為 0.0382%(σ=0.2)至 -0.0167%
(σ=0.5)——量級落在 50-seed 蒙地卡羅雜訊底以內(符號都不穩定,不構成
可判讀的系統性偏差),遠低於任一容差門檻(metal-shield 20%、2×1 OCL 15%、
dual-PD 10%)——可忽略不計。**

**不建議把預設 fitter 由 ols_forward(v1)全面換為 ols_inverse。** 理由:
純反向雖能把偏差方向修正、量級縮小(σ=2.0 時 1.37% vs 前向 2.30%),但其
變異(CV)始終劣於等權前向與 WLS,導致綜合 RMSE 在全部 4 個測試 σ 下都是
四者中最差(或並列最差,與 deming 逐列相同)。單純追求「消偏」而換
ols_inverse 反而讓整體精度變差。

**建議之條件式啟用路徑:**
- σ 落在典型預算(≤0.5)內:維持預設 ols_forward,偏差可忽略,無需切換。
- 若確認實際 disparity 噪聲較大(σ ≥ 1,例如低品質 SAD 區塊或
  metal-shield 稀疏取樣導致的區塊),建議啟用 `wls_inverse`(γ 預設 1 即可,
  不必精確標定也已有 ~9% CV 改善且 RMSE 全域最佳),而非切換到無權重的
  純反向。
- γ 對誤標不敏感(§3.3),故即使 qsigma 標定的 γ̂ 有系統性偏低
  (≈0.8 而非理論 1.0),仍可放心使用預設 γ=1 或標定值,不影響 WLS 消
  變異的效益。

**真實 SAD quality 到位後的標定步驟**(2026-07-18 起有現成命令與示範資料:
`dcc_cal --seq <序列> --qsigma`;示範 `data/examples/demo_fitter_wls_seq.json` +
step-by-step 見 `docs/fitter實驗_投影片.html` p.13–16):
1. 用實際序列跑 `dcc_cal --qsigma` 取得 p̂、σ̂₀、r²、bins,檢查 r² 是否
   足夠(多序列匯總參考基準 r²=0.9789;單一序列 ~480 樣本時 r²≈0.7–0.8 屬正常,
   r²<0.5 或 p̂≤0 表示 quality 與 σ 無穩定冪律關係,不應上 WLS 權重)。
2. 依 γ̂ = 2p̂ 換算權重指數(本次合成資料下 γ̂≈0.8,仍在敏感度分析證實
   的安全範圍內)。
3. 將標定結果(σ、γ)寫回 config 的 `regression.fitter=wls_inverse` 與對應
   γ 值;若噪聲量級落在典型預算內則維持預設 forward,不必啟用。
