1Sim 在系統中的定位

Sim(dcc_sim)是與外部 SAD 模組平行的「資料生產者」: 它產出的序列與外部模組交付的 disp_seq.json(SPEC-004 §3a)格式完全相同, 走同一個入口(讀取器 D-1〜D-6 驗證)進入管線——管線無法分辨資料來自 Sim 還是實機, 這是刻意設計:開發期用合成資料把「消費端」做好,對接時外部模組照格式即插即用。

GUI 中所有 Sim 參數變更走「dirty → 重生成 → 重跑管線 → 更新快照」閉環, 生成 48 區 × 10 幀為微秒級,體感即時。

三個資料面共用形狀:外層 = 幀(掃描點)、中層 = 列 r、內層 = 行 c(row-major)。 存檔檔名編碼生成條件(disp_seq_s0.50_b0.30_sd7_fc420_8x6_null3_<時間戳>.json), 內容採「每列一行」結構化縮排,任何編輯器直接可讀。

2.2 示範 RAW(UI 底圖,不入計算)

4608×3456 uint16(10-bit,值域 0..1023),無檔頭 little-endian。像素模型: 黑位 64 + 垂直線紋理(兩條互質週期弦波 37.0 / 61.7 px,避免與 16 px 取樣節拍共振)× 輕微暗角(−22% 角落)+ ±3 抖動;PD 位置(32×32 週期、L/R 各 8 點)施加 metal-shield 遮光(黑位以上衰減至 45%)。

3生成模型總覽(核心公式)

令 u = dac − fc(離焦量,DAC)、H = sweep 半幅(動態推導,§4.8)、 k = true_dcc(r,c)(該區 DCC 真值):

(3-1) disparitydisp(f,r,c) = (u/k) · (1 + nl2·(u/H) − nl3·(u/H)²) + bias + N(0, σ) [raw_pixel]
(3-2) focusfv(f) = 1000 · exp( −( (dac − (fc + peak_offset)) / H )² ) (全區同型)
(3-3) 真值場true_dcc(r,c) = center_dcc + (corner_dcc − center_dcc) · min(1, √(dx²+dy²)/√2)

其中 (3-3) 的 dx、dy 為 cell 對 grid 中心之正規化偏移(中心 0、四角 1)。 每一項對應 GUI 的一支滑桿;雜訊由 mt19937(seed) 驅動——同參數必出同資料 (bit-exact 重現性,IT-06 的基礎)。null 注入清單直接把指定 (幀,r,c) 輸出為 JSON null。

4參數逐一解析

4.1 dacs — 掃描序列(非 Sim 參數,由 config 推導)

dac_f 是第 f 幀的 VCM 馬達命令值,語意 =「這幀是在此鏡頭位置拍的」。 推導鏈(SPEC-002 C-1,禁止寫死):

span_af = MACRO − INF = 400 → FAR = 220 − 400×0.1 = 180;NEAR = 620 + 400×0.1 = 660
step = 480/(N−1) ≈ 53.33 → dacs = [180, 233, 287, 340, 393, 447, 500, 553, 607, 660]

(dac_f − fc) 即該幀離焦量——除以 true_dcc 得視差,正是 DCC 定義的反向運用。 點數 N = num_positions(5..50,預設 10)已參數化,交叉約束見 §8。

4.2 focus_center(fc)— 合焦位置 [DAC]

disparity 的零交叉點 = 回歸截距 b 的真值(預設 420)。模擬「chart 擺放距離」: 移動 fc = 模擬 chart 距離改變。滑到 E-F01 邊界(FAR+step, NEAR−step)之外 (如 640)→ focus 峰值出界中止。

4.3 center_dcc / corner_dcc — DCC 真值場

定義式 (3-3),中央 12.46、角落 14.5(假設模組期望值)。管線還原出的 48 區 DCC 應逐區等於此場(無雜訊時至 1e-9)——這就是 IT-01 的驗收基準。 把 center 設為負值(診斷選單「E-E01」)= 模擬 L/R 定義顛倒 → 回歸斜率非正。

4.4 noise_sigma(σ)— disparity 高斯雜訊 [raw px]

逐點獨立 N(0,σ)。與系統精度的關係(SPEC-005 §3a 誤差預算):

σ_k/k ≈ σ_d / √Sdd,√Sdd = √Sxx/k ≈ 38.9 raw px(預設 config)
→ σ = 0.5(外部驗收上限)→ DCC 波動 ≈ 1.3%;NFR-02(CV<2%)→ σ_d ≤ 0.78

注意:σ 推不動 err(§6)、也不改 DCC 期望值——它的殺傷力在重複性(CV)。

4.5 bias — disparity 系統偏差 [raw px]

常數偏移:只平移截距 b(Δb = k·bias),完全不動斜率 DCC。 bias=0.3(驗收上限)→ b 偏移約 3.8 DAC → err 貢獻僅 0.8%。用於驗證管線對 系統性 disparity 偏移的耐受性(IT-07 蒙地卡羅情境之一)。

4.6 nl2 / nl3 — 非線性(詳見 §5)

nl3(S 型壓縮)是 canonical 主導項;nl2(不對稱)為次要項。正規化到 H 使其語意 =「掃描端點處的效應比例」:nl3 = 0.2 → 端點視差幅度壓縮 20%。

4.7 focus_peak_offset / field_curvature / focus_amp_falloff — focus 峰值場

三者共同決定逐區 focus 峰值:peak(r,c) = fc + focus_peak_offset + field_curvature · radial(r,c)。

  • focus_peak_offset(全區均勻)— 模擬 PD vs 對比合焦之整體分歧;err 演練旋鈕(詳見 §6)。
  • field_curvature(逐區徑向)— 模擬場曲:各區合焦位置不同 → err heatmap 呈中央小、 角落大之徑向圖樣(實機 metal-shield 校正之主要判讀對象)。
  • focus_amp_falloff — 角落曲線振幅衰減(離軸 MTF/vignette);僅改外觀高低,不動峰值位置與 err(峰由 argmax 決定)。

三個 err 實機來源(場曲/chart 傾斜/PD 分歧)之詳細圖解另見 docs/focus曲線圖解說明.html §3.5。

4.8 half_span(H)— 正規化尺度(內部推導,非滑桿)

H = (dacs.back − dacs.front)/2,預設 config = 240。 它讓 (3-1)(3-2) 中的 u/H 在掃描端點恰為 ±1:nl 語意錨定於端點、focus 曲線在端點 落至峰值 1/e(整條 sweep 十個採樣點都有曲率資訊,四階多項式擬合穩定)。 沿革:曾寫死 240,已改為由 dacs 動態推導——margin 或 N 改動時語意自動跟隨。

4.9 null_cells / null_frames — 無效樣本注入

指定 (幀, r, c) 輸出 JSON null(= 外部 SAD 品質判定不合格)。GUI 快速鈕固定打 角落區 (5,7) 前 N 幀:N=2 → 該區有效 8 幀,剔除生效仍 PASS;N≥3 → 有效 7 < min_valid_samples(8)→ E-D03 中止。

4.10 fine_grid — 144×108 細粒度輸出

以 pattern block 粒度(32×32 px/block)輸出,行使讀取器 D-5 聚合 (median / quality 加權,cell 中心歸區、邊界歸左/上)。無雜訊時聚合結果與直接 8×6 一致(<1e-9,UT-10)。

4.11 unit / pitch_x / seed

  • unit:出檔前依宣告單位輸出(pd_image_grid 時 ÷16);讀取端是全案唯一入向轉換點 ——UT-03 驗證兩種單位輸入結果 bit-exact 一致。
  • seed:雜訊確定性之源。固定 seed = 同資料 → 報告 bit-exact(IT-06); 掃 seed 1..10 = 蒙地卡羅(IT-07)。

5非線性:鐘型 vs S 型(圖解)

視差對離焦的偏離有兩種本質不同的形態,對應 (3-1) 的兩個修正項:

nl3 — S 型壓縮(canonical,主導)nl2 — 不對稱(次要)
數學殘差 ∝ −u³(奇函數)殘差 ∝ +u²(偶函數)
形態兩端反向偏離 → S 型;幅度對稱壓縮兩端同向偏離 → 鐘型
物理來源PD 像素角度響應飽和、模糊圈超出相關窗、光瞳裁切球差等造成的近端/遠端不對稱
對稱性判別disp(−u) == −disp(+u)(測試逐 bit 釘死)奇對稱必然破壞
ΔDCC vs 合焦偏移 Δ偶函數(±Δ 等效,皆使 DCC 偏高)線性反對稱(≈ 2·nl2·Δ/H)
沿革註記(2026-07-13) 初版僅有偶次 nl(當時觀察到兩端同向飄移),經工程師比對校正手冊圖示(S 型)提出質疑, 確認偶次項只涵蓋不對稱效應、canonical 主導應為奇次壓縮 → 升級為 nl2+nl3 雙項模型, 並以「奇對稱逐 bit 判別式」測試釘死兩者不可混淆。

6err 機制與 focus 峰值偏移

Phase F 的判定量 err = |b − focus_peak| / span 度量「兩條獨立合焦估計」的分歧。 Sim 中兩者的真值分別為:

回歸截距 b → fc(disparity 零交叉點) focus 峰值 → fc + peak_offset
∴ err = |peak_offset| / span(精確關係;offset 96 → err = 96/480 = 0.20 恰踩規格線)

「focus 曲線與峰值」面板的逐元素圖解(來源鏈、參數對應、看圖識因)另見 docs/focus曲線圖解說明.html。

常見疑問:「為什麼雜訊 σ 拉滿也很難讓 err 變紅?」 這不是 bug,是設計。雜訊打到截距的力道經 √n 稀釋: σ_b ≈ k·σ/√n → σ=2 時 σ_b ≈ 8 DAC → err 波動僅 ≈ 1.7%,48 區最差 ≈ 4% ≪ tolerance 0.20。 err 的職責是抓系統性分歧(用 peak_offset 演練),隨機雜訊的殺傷力由 NFR-02(重複性 CV)把關——兩個機制各司其職。
