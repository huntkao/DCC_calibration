2輸入契約:disp_seq.json

你的上游要產這個檔(v1,已凍結)。彈性=你可以變、鎖定=契約不可違反:

欄位內容約束 / 彈性

pitch_x節距(驗算用)須等於 config 推得值(範例 16)

3核心算法:為什麼輸出可信

每一區(共 48 區)把 N 個掃描點做線性回歸 DAC = k·disparity + b:斜率 k = DCC(要燒的值),截距 b = 合焦點。閉式解、無迭代、確定性——同輸入必得同輸出(離線可 bit-exact 重算)。

  • 交叉驗證:截距 b 說的合焦點,要跟 focus 曲線峰值對質;逐區 err = |b − focus_peak| / sweep_span 全部 ≤ 容差才 PASS,否則 FAIL 附最差三區座標。
  • fitter 可換:預設 ols_forward 在典型噪聲(σ≤0.5)下偏差可忽略;若上游噪聲大(σ≥1),config 改 wls_inverse 即可,不動程式。
  • 防護:區內有效樣本 < 8 → E-D03;解出 k ≤ 0(幾乎都是 L/R 對調)→ E-E01。

4單位契約:16 倍陷阱(對接最關鍵)

下游單位還沒定案也沒關係 runtime PDLIB 要哪種單位由 config output_disparity_unit 決定——定案那天翻一個值即可,校正結果不用重跑程式。

6config 彈性速查:情境 → 動哪個

當上游條件變了你只需改
SAD 換輸出粒度(如 144×108)序列自帶 grid + aggregation.method / min_valid_ratio
掃描點數加密/減少dcc.num_positions(5..50)
視差單位不同dcc.input_disparity_unit
下游 PDLIB 單位定案dcc.output_disparity_unit
上游噪聲偏大(σ≥1)regression.fitter=wls_inverse + weight_gamma
換 sensor 容差等級dcc.tolerance(0.20 / 0.15 / 0.10)
EEPROM 精度取捨dcc.q_format(4..8)
掃描窗端點vcm.af_cal_inf/macro + dcc.far/near_margin(端點由公式推導,不寫死)

原則:所有可調參數都在 config,不散落程式常數。刻意不可調:掃描方向(VCM 磁滯)、core 內單位(raw_pixel 唯一)——這兩處是設計面鎖定。

7錯誤碼速查(對接除錯用)

碼意義對接時先查
E-A01AF 校正值缺失/非法config 的 vcm.af_cal_inf/macro
E-C01sweep 超出 DAC 範圍margin 或 AF 值
E-D01序列維度/形狀閉環驗算失敗disp_seq 的 N / grid 與 config 對不上
E-D02DAC/unit/pitch 與規劃或 config 不一致外部模組輸出 vs config 單位/節距/dacs
E-D03區域有效樣本不足chart 覆蓋不足 / 髒污 / 局部過曝
E-E01回歸斜率非正LEFT/RIGHT 影像顛倒
E-F01focus 峰值出界chart 距離擺放 / 場曲
E-G01Q-format 溢位q 值與 DCC 量級不匹配
失敗現場可重放 任何中止都會落盤 abort_dump.json(序列 + 參數快照)+ 雙 sink log。一筆失敗就是一筆可離線重跑的測資,回報問題時附上這個目錄即可。
