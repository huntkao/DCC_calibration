PDAF Module Calibration · Production Line Procedure

DCC 校正 Step-by-Step 詳解

以 Metal-Shield(遮光式)PDAF 感光元件為例 · 依 Qualcomm 80-NV125-1 流程展開 · 所有數值為假設的教學範例

SPEC本文假設條件

DCC 的目標只有一件事:量出「鏡頭要移動多少 DAC,才能消除 1 pixel 的相位視差」。以下用一組具體的假想模組貫穿全文。

項目假設值
SensorSparse metal-shield PDAF,解析度 4608 × 3456,10-bit RAW(0–1023 LSB)
PD pattern每 32 × 32 pixel 區塊內含 8 對 左遮光(L) / 右遮光(R) G 像素
VCM 馬達10-bit DAC(0–1023),數值越大鏡頭越靠近 Macro(正向致動器)
AF 校正結果AF_CAL_INF = 220 DAC(無限遠合焦)、AF_CAL_MACRO = 620 DAC(10 cm 合焦)
Margin非望遠鏡頭 → DCC_LP_FAR_MARGIN = DCC_LP_NEAR_MARGIN = 0.0
DCC map影像切 8 × 6(寬 × 高)= 48 區塊,每區塊一個 DCC 值
驗證容差Metal-shield 型預設 20%(DCC_VAL_TOL_SPARSE)

PHASE A前置條件確認

  1. AF 校正已完成 本模組量得 AF_CAL_INF = 220 DAC、AF_CAL_MACRO = 620 DAC。DCC 校正的掃描範圍完全由這兩個值推導,沒有 AF 校正結果就不能做 DCC。
  2. Gain Map(SPC)已完成並可套用 PD 像素經過 L/R gain map 補償後,平場下 L 與 R 訊號位準才會一致;否則視差計算會被亮度差污染。
  3. Sensor configuration 已核對 PD 像素座標(依 x 遞增排序)、pattern 週期 32×32、影像為 canonical orientation(flip/mirror 關閉)。L/R 標記若顛倒,算出的 DCC 會是負值,工具直接回報校正失敗。
  4. OIS(若有)已校正並鎖定於參考 XY 位置 確保鏡頭 CRA 特性符合實際使用情況,否則 DCC map 形狀會偏。

PHASE B測試場景架設

B-1 · 計算 chart 擺放距離

Chart 要放在「鏡頭位於掃描範圍正中間時剛好合焦」的物距上:

中間鏡頭位置 = (AF_CAL_INF + AF_CAL_MACRO) / 2 = (220 + 620) / 2 = 420 DAC 查本鏡頭的 DAC ↔ 物距對照 → 420 DAC 合焦物距 ≈ 25 cm

所以 chart 面板中心與模組入瞳的距離設為 25 cm(一般模組落在 20–30 cm;望遠鏡頭可能到 2 m)。

B-2 · 決定 chart 尺寸與型式

使用垂直線 chart(優先,視差估測比 diamond chart 均勻)。尺寸原則:相機 FOV 覆蓋 chart 有效區的 85%–95%。假設水平 FOV = 78°,在 25 cm 處視野寬約 2 × 25 × tan(39°) ≈ 40.5 cm,則 chart 有效區寬度約 43–48 cm。

B-3 · 光源與曝光設定

項目設定檢查方式
光源D50 / D65,照度 ≥ 400 lux照度計實測 chart 面
Sensor gainanalog = 1X, digital = 1X(metal-shield 型)暫存器回讀
曝光chart 白底對應的中央 10%×10% ROI 內,L、R 像素平均值各落在 300–950 LSB(10-bit)抓一張 RAW 統計
一般像素不得飽和直方圖確認無 1023 峰
Frame averaging視差重複性差時啟用多張平均同位置重拍比對
為什麼一般像素不能飽和? 遮光像素靈敏度低,鄰近的一般像素若飽和,電荷會溢流(blooming)進遮光像素,直接污染視差值。這是 metal-shield 型最常見的隱性誤差來源。
Chart 五大 NG:太小(背景入鏡)、太大(線條過粗於視差搜尋窗)、旋轉、傾斜(左右物距不等)、過曝。任何一項都會造成系統性 DCC 偏差,量再多次也救不回來。

PHASE C鏡頭掃描與影像擷取(10 個位置)

C-1 · 計算掃描範圍與步距

FAR_LP_LIMIT = AF_CAL_INF − (MACRO−INF) × DCC_LP_FAR_MARGIN = 220 − 400 × 0.0 = 220 DAC NEAR_LP_LIMIT = AF_CAL_MACRO + (MACRO−INF) × DCC_LP_NEAR_MARGIN = 620 + 400 × 0.0 = 620 DAC LP_STEP_SIZE = |620 − 220| / 9 = 44.44 DAC (共 10 個取樣點)

C-2 · 逐張擷取流程

  1. 鏡頭移到 #1(220 DAC) 下 DAC 指令後等待鏡頭穩定(VCM 典型 settle time 10–30 ms;彈簧振鈴未停就拍會抖出視差雜訊)。
  2. 拍攝全解析度 RAW Sparse PDAF 必須給全幅 RAW(4608×3456,10-bit),PD 像素的解析由校正 DLL 依 sensor configuration 進行,不要自己先抽。
  3. 呼叫 PDAF_Cal_add_raw(...) 入堆疊 傳入影像指標、寬高、black level(如 64 LSB)、bit depth、目前鏡頭 DAC 值。檢查回傳碼——飽和、曝光不足、chart 偵測失敗等任一錯誤都必須立刻中止。
  4. 前進 44.44 DAC 到下一位置,重複 1–3 實務上 DAC 取整數:220, 264, 309, 353, 398, 442, 487, 531, 576, 620。方向單向由 FAR 到 NEAR,避免 VCM 磁滯(hysteresis)讓同一 DAC 對應到不同實際鏡頭位置。
  5. 第 10 張(620 DAC)入堆疊後,擷取階段結束 堆疊內現在有 10 張 RAW + 10 個對應 DAC 值。

PHASE D相位視差計算(DLL 內部,理解用)

這一段是 PDAF_Cal_get_dccmap(...) 內部對每一張影像做的事。開發演算法時,你的 runtime PDLIB 也是同一套邏輯。

  1. 解析 PD 像素 → 產生 L / R 兩張低解析影像 依 sensor configuration 從全幅 RAW 抽出所有 L 與 R 像素。以 32×32 pattern、每 pattern 8 對計算:每列 4608/32 = 144 對,共 3456/32 = 108 個 pattern 列 → L/R 影像各約 144 × 216(依 pattern 內配置而定)。
  2. 套用 Gain Map 補償 L 影像乘上左 gain map、R 影像乘上右 gain map(13×17 內插到 PD 影像尺寸),消除遮光造成的 shading 與 L/R 靈敏度差。
  3. 切成 8×6 區域 每區約 576 × 576 原始像素(對應 PD 影像上約 18 × 36 個 PD 樣本)。之後所有計算都逐區獨立進行。
  4. 區域內做 1D 水平位移搜尋 把 L、R 訊號沿水平方向以 SAD(絕對差和)在 ±搜尋窗內滑動比對,SAD 最低點附近再做次像素內插 → 得該區的 phase disparity(單位:pixel,含正負號)。
10 張影像 × 48 區域 = 480 個 disparity 值,加上 10 個 DAC 位置,就是 Phase E 回歸的全部原料。同時 DLL 也對每張影像逐區計算 focus value(對比度銳利度),留給 Phase F 驗證用。

PHASE E線性回歸求 DCC

以中央區域(region 3,3)為例,10 個位置量到的 disparity 如下(假設數據):

#DACdisparity (px)#DACdisparity (px)
1220−16.16442+1.7
2264−12.47487+5.3
3309−8.98531+8.8
4353−5.29576+12.5
5398−1.810620+16.0

對 (disparity, DAC) 做最小平方直線擬合 DAC = k·disparity + b:

斜率 k = ΔDAC / Δdisparity ≈ (620−220) / (16.0−(−16.1)) = 400 / 32.1 ≈ 12.46 DAC/pixel → DCC(region 3,3) 截距 b ≈ 421 DAC → disparity = 0 時的「PDAF 預測合焦位置」
48 個區域各自回歸 → 得 8×6 DCC map。因場曲與 shading,周邊 k 值通常高於中央(例:中央 12.5,四角 14–15 DAC/px)。健康的 DCC map 是平滑曲面;若出現大量局部凹凸,代表某些區域視差估測失敗,必須查根因(chart 覆蓋不足、髒污、局部過曝)而不是硬燒。
DCC 符號檢查:QTI 定義 DCC 為無號正值。若回歸斜率為負,DLL 回報錯誤——幾乎都是 sensor configuration 的 LEFT/RIGHT 標反了,回 Phase A 修正。

PHASE F整合驗證(同一次掃描完成)

掃描時每張影像已同步算出 focus value(對比銳利度),用它交叉驗證 DCC:

  1. 對 focus 曲線做多項式擬合,取峰值位置 假設中央區域 focus 峰值落在 427 DAC,視為「真實」合焦位置。
  2. 與 phase 直線的截距比較 PDAF 預測 b = 421 DAC,誤差 |427 − 421| = 6 DAC。
  3. 以掃描行程正規化,對照容差 6 / 400 = 1.5% ≪ metal-shield 容差 20% → PASS。48 區逐一判定(容差同為 8×6 grid)。
  4. 檢查 focus 峰值不在掃描端點 若峰值卡在 220 或 620 DAC 端點,DLL 回報 0x2000: Focus peak is out of boundary——常見於 chart 距離錯誤或望遠鏡頭場曲 >10% 未先篩除。

PHASE G寫入 EEPROM / OTP

  1. 取得燒錄 buffer 呼叫 PDAF_Cal_get_calibration_block(...),依 Calibration Format v4 打包:版本號 → 左/右 gain map(各 221 點)→ DCC Q format → 8×6 DCC map(48 值,各 2 bytes,H/L)→ Checksum %256。
  2. DCC 以 Q format 定點數儲存 例:Q6 格式下 12.46 DAC/px 存為 round(12.46 × 64) = 797 = 0x031D。
  3. 燒錄後回讀驗證 比對 buffer 與 checksum;回讀失敗的模組不得流出。
  4. Runtime 使用 PDLIB 開機讀入 gain map 與 DCC map。對焦時:量得某 ROI 的 disparity(例 −4.0 px)→ 查該 ROI 的 DCC(12.46)→ 鏡頭移動 −4.0 × 12.46 ≈ −50 DAC → 一步跳到預測合焦位置,再由 Contrast AF 做最後微調。
整條流程回顧:架 chart(25 cm)→ 220→620 DAC 掃 10 點 → 每點抽 PD、補 gain、逐區算 disparity → 48 區各做直線回歸取斜率 = DCC map → 以 focus 曲線峰值驗證(誤差 < 20%)→ 打包燒錄 EEPROM。單顆模組產線節拍(含 10 次移動+拍攝+運算)典型在 2–4 秒內。
