這張圖不是單一資料的呈現——它把三個不同階段的產物疊在同一座標系, 讓「合焦在哪(Sim/實機)、量到哪(Phase F)、允許到哪(config)」一眼對齊:

折線 + 10 個樣本點Sim 生成 → Phase D 載入該區的 fv[f] 原始值(序列檔 focus 資料面);x 座標 = dacs(掃描序列)

3Sim 工作台參數對應

參數 / 動作對這張圖的影響
合焦位置 fc整條曲線(連同峰)水平平移;滑到 640 → 峰入紅區 → E-F01 中止(診斷選單「E-F01」即此)
focus 峰值偏移曲線峰移至 fc+offset,但 b 留在 fc → 峰值線與 b 線分離、err 上升;±96 → E-F02(診斷選單「E-F02」)
config margin / num_positions決定樣本點位置與數量;H 為動態半幅 → 曲線寬度跟著 sweep 變(端點恆為峰值 1/e);紅區邊界同步連動
config focus.poly_orderPhase F 擬合階數(預設 4);樣本數須 > 階數(loader 交叉約束)
config focus.peak_margin_steps紅區寬度 = m×step;約束 2m < N−1(否則有效窗為空,loader 拒絕)
144×108 細粒度fv 同走 D-5 聚合(Sim 之 fv 全區同值,聚合後不變)
刻意「不」影響它的參數(常見誤會點) σ、bias、nl2、nl3、null 注入都只作用於 data(disparity)面,不碰 focus 面—— 所以拉大雜訊時回歸散點變毛,這張 focus 曲線卻依然光滑。這是目前 Sim 的簡化 (實機的 fv 會有雜訊、無效幀通常兩面一起缺);M2 對接實測後若需演練 fv 雜訊對 峰值擬合的影響,再為 focus 面增設獨立注入(4 階擬合本身有平滑效果,預期耐受性不差)。

4快速判讀表(看圖識因)

看到的現象原因 / 對應機制
peak 線與 b 線分離系統性合焦分歧(Sim:peak_offset;實機:場曲/chart 傾斜/PD vs 對比合焦差)→ err 上升

曲線平移但 peak 與 b 一起動、err 不變只動了 fc(chart 距離)且未觸界——兩條估計同源移動,交叉驗證本來就不該報警
