3.5err 的三個實機來源(把「場曲/chart 傾斜/PD 分歧」講清楚)

err 量的是「disparity 說的合焦點(截距 b)」與「對比度說的合焦點 (focus 峰值)」的分歧。實機上讓這兩者分開的三個原因,空間特徵各不相同:

來源是什麼空間特徵 / 對應 Sim 參數
① 場曲
(field curvature)		鏡頭的合焦面不是平面而是微彎碗狀(Petzval 像差)。中央清晰時角落還糊,要再推一點鏡頭角落才清——每一區的真實合焦位置本來就不同,是鏡頭本身的光學特性。徑向(中央小、角落大)
field_curvature
② chart 傾斜
(chart tilt)		圖卡沒和光軸垂直、歪了 → 圖卡不同部位到相機的物理距離不同,上下緣落在不同合焦距離。是治具/擺放的幾何問題,不是鏡頭。跨畫面線性漸變
(Sim 未建模,治具端控制)
③ PD vs 對比合焦差
(最關鍵)		「disparity=0(PD 說合焦)」與「影像實際最銳利(對比峰)」系統性差一點:PD 被 metal-shield 遮半、只用部分光瞳(不對稱孔徑),其角度/光譜響應與全孔徑成像不同;disparity 零點對應「PD 眼中的合焦」,未必等於全孔徑 MTF 最高點。這正是 DCC 交叉驗證要抓的系統誤差——太大則一步跳焦會落在銳利點之外。通常向角落惡化
(可用 field_curvature 疊加近似)
實機各區 focus 曲線會一樣嗎?——不會 目前若只設全區均勻的 focus_peak_offset,48 區是同一條曲線;但實機至少三處逐區不同:
  • 峰值位置不同 ← 場曲(①)。這是最重要的:正因各區合焦位置不同,實機 err 才會有 徑向圖樣(中央綠、角落偏黃/紅),而非被單一 offset 整片推動。用 field_curvature 重現。
  • 峰值高低/寬窄不同 ← 離軸 MTF 下降 + 角落 vignette 進光少 → 角落曲線較低較平 (峰不明顯、擬合較不穩)。用 focus_amp_falloff 重現(僅外觀,不改 err)。
  • 雜訊/紋理能量不同 ← 角落訊噪比差、圖卡紋理不均。目前 Sim 尚未對 focus 面加雜訊(見上方警示框)。
換言之,metal-shield 校正真正要判讀的就是這個「中央好、角落差」的 err 徑向圖樣, 而它的容差設 20%(比 dual-PD 10% 寬)正因遮半 PD 先天分歧較大。
