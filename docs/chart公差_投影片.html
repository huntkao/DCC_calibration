兩個獨立誤差源,經 nl 這個傳導係數耦合

環節物理成因既有框架
① 曲線非線性球差 / PD 響應飽和(nl)--scan
② chart 距離誤差治具擺放幾何誤差(cm)chart_dist(本次)
  • ①「合焦偏移(DAC)→ DCC 誤差」已由 --scan + synth nl 驗證。
  • ②「chart 物理距離(cm)→ 合焦偏移(DAC)」原本是文字工序,程式零實作 —— 本案補上。
  • 因果關鍵:chart 距離公差本身不是 nl 的成因,二者是平行的獨立誤差源, 只有透過 nl 這個傳導係數耦合才會顯現為 DCC 誤差。
因此換算鏈路必須雙向串接 ①+②:cm 偏移 → DAC 偏移 → ΔDCC%, 才能回答「chart 治具要準到多少 cm」。

VCM 薄透鏡近似:DAC 與 1/物距 呈線性,兩點標定求解

b = (dac₁ − dac₂) / (1/dist₁ − 1/dist₂)    a = dac₁ − b / dist₁

上線用法

Step-by-step:從實機量測到產線治具公差規格

1. 量測非線性 nl 外部:球差 / PD 響應飽和,實模組量測,無預設值可代替
2. 取 VCM 兩點標定 該鏡頭 INF / MACRO 的 DAC↔物距對照點,填 --vcm-cal
3. 跑 --chart-tol dcc_cal --dry-run --chart-tol --nl <實測> --vcm-cal <實測> --dcc-budget <允收>
4. 讀反算行 得治具擺放公差 ±cm,交產線治具規格

開放問題 #3 狀態

依 2026-07-17「離線+實驗、不碰硬體」定位,真實 nl 與 VCM 標定屬外部承擔—— 本工具的角色是把「兩個獨立誤差源」正確串接成可雙向換算的工具,不是提供某一鏡頭的最終公差數字。

關閉條件:換算工具(正/反算 + 兩點標定 + CLI)備妥,即視為 SPEC-005 §7 #3 關閉——不等待真實量測數據到位。
