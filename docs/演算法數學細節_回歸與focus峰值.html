1全景:一條管線,兩具擬合引擎

每一個影像分區(預設 8×6 = 48 區)都要獨立跑兩次擬合,兩者互為交叉驗證:

為什麼「同一個合焦點要算兩次」 回歸的截距 b 是相位視差外推出的合焦位置(disparity=0 對應的 DAC); focus 峰值是對比清晰度直接量測的合焦位置。兩者理論上應相等—— 不相等的量就是校正誤差 err,這正是交叉驗證的核心(SPEC-002 Phase F)。

2回歸引擎:普通最小平方(OLS)閉式解

實作:src/dcc_core/src/regression.cpp · fit_region()

2.1 模型與待解量

單區內,沿掃描 10 站得到成對樣本 (xᵢ, yᵢ),其中 x = disparity [raw_pixel]、 y = DAC。假設線性關係:

y = k · x + b   // k = DCC 斜率 [DAC/raw_pixel](恆正)、b = 截距 [DAC](= 合焦點)

目標:找 k, b 使殘差平方和最小。

S(k, b) = Σᵢ ( yᵢ − k·xᵢ − b )²

2.2 從一階條件導出閉式解

對 b 與 k 各取偏導令零(正規方程):

∂S/∂b = −2 Σ( yᵢ − k xᵢ − b ) = 0  ⇒  b = ȳ − k·x̄
∂S/∂k = −2 Σ xᵢ( yᵢ − k xᵢ − b ) = 0  ⇒  k = Sxy / Sxx

其中 x̄, ȳ 為均值,分子分母是中心化(去均值)後的交叉積與平方和:

Sxx = Σ(xᵢ − x̄)²    Sxy = Σ(xᵢ − x̄)(yᵢ − ȳ)    Syy = Σ(yᵢ − ȳ)²

2.3 單趟累加的計算形式(實作採用)

程式不先算均值再減——那要掃兩趟。改用代數等價的單趟累加:一次迴圈收齊 Σx, Σy, Σx², Σxy, Σy² 五個原始和(NaN 樣本略過、不計入 n),再用下式還原中心化量:

Sxx = Σx² − (Σx)²/n    Sxy = Σxy − (Σx)(Σy)/n    Syy = Σy² − (Σy)²/n

於是三個輸出量:

k = Sxy / Sxx     b = (Σy − k·Σx) / n     R² = Sxy² / (Sxx·Syy)

其中 b = (Σy − k·Σx)/n = ȳ − k·x̄,與 2.2 一致;R² 為判定係數,量測擬合品質 (< r2_warn 時上層發警告,不中止)。

數值細節:disparity 無變異的退化 若該區 10 站的 disparity 幾乎不動(例如場景/區域無足夠紋理),Sxx ≤ 0, 斜率無定義 → 擲 E-D03。這比讓 k 變成天文數字後才爆更早、更好診斷。

2.4 為什麼 v1 用 OLS,而非正交回歸

OLS 假設只有 y(DAC)有誤差、x(disparity)無誤差。實際上 disparity 由外部 SAD 模組 量出,本身帶噪(σ_d),屬 errors-in-variables(EIV)情形,OLS 斜率會有已知的衰減偏差 (往零方向偏)。M1 先用 OLS 打通管線,並在 M0 誤差預算(M0_階段報告 §8)量化了這個偏差量級; fit_region 的介面刻意讓 fitter 可抽換,EIV 修正列為 M2 備忘(SPEC-005 §8 #2)。

2026-07-15 起,這條升級路的前置已就位:每樣本 σ 的來源有了定義——外部序列的 quality 面(SPEC-004 §3a.1 語意提案:q ∝ 1/σ²)可直接推 WLS 權重或 EIV 之 σ_x; Sim 的 focus_linked 模型以 σ_eff = σ₀/√q 誠實掛鉤生成驗收資料 (《Sim 工作台完全解析》§7)——M2 換裝 fitter 時,σ_k 的增益可以被量化驗收, 而非只能「相信理論」。注意:v1 的 OLS 不消費 quality; quality 目前僅在 D-5 聚合(引擎上游)作 weighted_mean 權重。

3focus 峰值引擎:多項式擬合 + 密集掃描

實作:src/dcc_core/src/focus.cpp · peak() / solve_linear()

focus 曲線(清晰度 vs DAC)呈單峰鐘形,峰頂即對比合焦位置。取峰值分三步: 座標正規化 → 最小平方擬合多項式 → 密集掃描取最大。

3.1 座標正規化(關鍵的數值前處理)

DAC 絕對值落在數百量級(例:180–660)。若直接以這種大數的高次冪建 Vandermonde 矩陣, x⁸ ≈ 660⁸ ≈ 3.6×10²²,與低次項相差二十幾個數量級,正規方程矩陣條件數爆掉、解不可信。 先把 DAC 線性映射到 [−1, 1]:

center = (dacmin + dacmax)/2    half = (dacmax − dacmin)/2    tᵢ = (dacᵢ − center) / half

擬合在 t ∈ [−1,1] 上進行(tⁿ 恆有界),最後再把峰值 t* 映射回 DAC: peak_dac = center + t*·half。

3.2 多項式最小平方 → 正規方程

擬合 P 階多項式(預設 poly_order = 4,係數 m = P+1 個):

p(t) = c₀ + c₁t + c₂t² + … + cPtP

最小化 Σᵢ(yᵢ − p(tᵢ))²。以 Vandermonde 設計矩陣 V(V[i][j] = tᵢ ʲ)寫成 正規方程:

(Vᵀ V) · c = Vᵀ y

逐元素即冪次和(Hankel 結構),實作正是這樣累加:

(VᵀV)r,c = Σᵢ tᵢ(r+c)      (Vᵀy)r = Σᵢ tᵢr · yᵢ    (r, c = 0…P)

每個樣本先算好 tᵢ⁰ … tᵢ²ᴾ 這 2m−1 個冪次,填入 m×m 矩陣與 m 維右端向量。

3.3 解線性系統:部分主元高斯消去

m 很小(5×5),用直接法即可。solve_linear() 做部分主元(partial pivoting) 高斯消去:

  1. 逐行選主元:在當前欄由對角起往下找絕對值最大的列,與當前列對調(降低捨入誤差、避免小主元放大)。
  2. 主元絕對值 < 1e−14 視為奇異(樣本退化)→ 擲 E-D03。
  3. 前向消去成上三角,再回代解出係數 c。

3.4 密集掃描取峰(而非解析求導)

拿到 c 後,不解 p′(t)=0(4 階多項式的導數是 3 次,要開根、還要辨別多個駐點裡哪個是範圍內全域最大)。 改用穩健的密集掃描:在 t ∈ [−1,1] 均勻取 4801 點,逐點以 Horner 展開求值,取最大者:

t* = argmaxt ∈ {−1 … 1, 4801 點} p(t)   ⇒   peak_dac = center + t*·half
4801 點 = ~0.1 DAC 解析度 以掃描範圍 480 DAC 為例:480 / (4801−1) = 0.1 DAC。這遠細於 VCM 的 DAC 量化步階, 次像素峰值已足夠精確;而密集掃描天然回傳範圍內全域最大,不會被邊緣的偽駐點騙到,程式碼也簡單無分支陷阱。

3.5 出界檢查(chart 距離/場曲防呆)

合焦峰值必須嚴格落在掃描範圍內側,且離兩端至少 peak_margin_steps 站(預設 1 站):

step = (dacmax − dacmin) / (N−1)    合格區間 = ( dacmin + margin·step ,  dacmax − margin·step )

峰值貼邊代表真正的合焦點可能在掃描範圍之外——通常是 chart 距離設錯或場曲把某區合焦推出範圍。 此時擲 E-F01,提示檢查 chart 距離,而非默默回傳一個不可信的邊界值。

4判定:截距 vs 峰值的一致性誤差

實作:src/dcc_core/src/validate.cpp · judge() / region_pass()

兩具引擎各給出一個「合焦 DAC」:回歸截距 b 與 focus 峰值 peak。逐區的一致性誤差:

err = | b − peak | / span    // span = 掃描全幅(dacmax − dacmin),把誤差正規化成無量綱比例
region PASS ⇔ err < tolerance    整體 PASS ⇔ 所有區皆 PASS

報告另取 err 最大的三區(排序後前三)供除錯定位(FR-17)。UI 誤差 heatmap 即以此 err 上色(深綠→黃),超差區塗紅。

為什麼用 span 正規化 不同模組/不同掃描設定的 span 不同,除以 span 讓 tolerance(如 metal-shield 20%)成為 跨設定可比的統一門檻,不必隨掃描範圍重調。

5數值穩定性與失敗碼一覽

失敗碼觸發條件數學/物理原因出處
E-D01dacs 與 disp/fv 長度不一致閉環驗算失敗(資料組裝錯誤)regression / focus
E-D03有效樣本 < 門檻;或 Sxx ≤ 0;或擬合矩陣奇異(主元 < 1e−14) 樣本退化,斜率/多項式係數無定義regression / focus
E-E01回歸斜率 k ≤ 0DCC 恆正;非正代表 LEFT/RIGHT 定義反了regression
E-F01focus 峰值出界(貼掃描端點)合焦點落在掃描範圍外——chart 距離錯或場曲focus

數值穩定性設計摘要

  • 回歸單趟累加:一次迴圈收 5 個矩,再還原中心化量,省一趟且與兩趟解代數等價。
  • 回歸中心化:用 Sxx=Σx²−(Σx)²/n 去均值,避免大數相減的災難性抵消(尤其 disparity 均值不為零時)。
  • focus 座標正規化到 [−1,1]:把 Vandermonde/正規方程的條件數從 ~10²⁰ 壓回可解範圍,是能用直接法解 5×5 的前提。
  • 部分主元:高斯消去選絕對值最大主元,控制捨入誤差;奇異即早停報 E-D03。
  • 密集掃描取峰:回傳範圍內全域最大,避開解析求導的多駐點辨識;0.1 DAC 解析度已優於 DAC 量化。
  • 邊界防呆:峰值貼邊直接 E-F01,不回傳不可信值——把「chart 沒擺好」變成明確錯誤而非隱性壞資料。
